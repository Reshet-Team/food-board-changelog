# reshet-template

An opinionated React starter template, used internally by the Reshet team.

## Stack

| Layer             | Technology                           |
| ----------------- | ------------------------------------ |
| Framework         | React 19 + Vite                      |
| Routing           | TanStack Router (file-based)         |
| Server state      | TanStack Query                       |
| UI primitives     | Base UI (`@base-ui/react`)           |
| Component library | Reshet UI                            |
| Styling           | SCSS Modules + CSS custom properties |
| Language          | TypeScript (strict)                  |
| Testing           | Vitest + Testing Library             |
| Package manager   | pnpm                                 |

## Quick Start

```bash
pnpm install
pnpm dev       # http://localhost:5173
```

## Key Conventions

### Routing

Routes live in `src/routes/` and are auto-discovered by TanStack Router. After adding or renaming a route file, the type-safe route tree is regenerated automatically during `pnpm dev`, or manually with `pnpm generate-routes`.

Never edit `src/routeTree.gen.ts` by hand.

### Feature slices

All product logic lives under `src/features/<feature-name>/`. A feature owns its own components, hooks, services (API/query definitions), and any local utilities. Cross-feature shared code goes in `src/hooks/`, `src/lib/`, or `src/utils/`.

### RTL & direction

The root layout wraps the app in `<DirectionProvider direction="rtl">` from Base UI. The default font is **Heebo**. If a project requires LTR, change the `direction` prop in `src/routes/__root.tsx`.

### Theming

`ThemeProvider` in `src/theme/useTheme.tsx` exposes `{ theme, setTheme, resolved }` via React context. It supports `"light"`, `"dark"`, and `"system"` and persists the preference to `localStorage`. The resolved theme is applied as `data-theme` on `<html>`.

Design tokens are plain CSS custom properties defined in `src/theme/` using the **OKLCH** color space and split into:

- `colors.scss` ‚Äî semantic color tokens (primary, danger, success, ‚Ä¶)
- `typography.scss` ‚Äî font family, size, weight, line-height
- `spacing.scss` / `sizes.scss` / `radius.scss` / `shadows.scss` / `transitions.scss`

### Path alias

`@/` resolves to `src/`. Use it for all non-relative imports.

```ts
import { Button } from '@/components/ui/Button/Button'
```

### Naming conventions

| What                          | Convention                       | Example                      |
| ----------------------------- | -------------------------------- | ---------------------------- |
| Component files & exports     | PascalCase, in a matching folder | `Button/Button.tsx`          |
| Component SCSS modules        | PascalCase, match the component  | `Button/Button.module.scss`  |
| Hook files                    | camelCase, `use` prefix          | `useGreeting.ts`             |
| Feature / folder names        | camelCase                        | `features/userProfile/`      |
| Route files                   | kebab-case                       | `user-profile.$userId.tsx`   |
| Utility & service files       | camelCase                        | `queryClient.ts`, `date.ts`  |
| TypeScript interfaces & types | PascalCase                       | `ButtonProps`, `UserProfile` |

## Adding Components with Reshet UI

Reshet UI is the Reshet team's shadcn-compatible component registry. Add a component with:

```bash
pnpx shadcn@latest add @reshet-ui/<component-name>
```

Components are written into `src/components/ui/`.
